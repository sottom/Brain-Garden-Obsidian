## Architecture & Integration

System Boundaries:

- Loyalty Engine (Centaurus team): Tracks year-to-date spend, tier promotions, calculates earn rates, triggers deposits/withdrawals to Ledger 222
- Loyalty Ledger (POS team): Maintains F&D buck and cashback account balances, processes deposits/withdrawals, provides balance APIs 27
- Sales feed provides redemption events to Loyalty Engine for reconciliation and fraud detection 2324

Event Flows:

- Loyalty Engine sends deposit events (F&D bucks earned, cashback accrued) to Ledger via API 2225
- POS handles tender-type redemptions; Loyalty Engine receives redemption data from sales feed for audit 2326
- Non-sales events (cashback payouts, failed ACH deposits, fraud flags) must be communicated back to Loyalty Engine for ledger reconciliation and marketing triggers 2728

Reconciliation:

- Daily event-by-event comparison: Loyalty Engine's ledger entries vs. POS Ledger entries streamed to BI team 1215
- Balance validation: Sum of all line items from both systems must match; detects SQL-level fraud 12
