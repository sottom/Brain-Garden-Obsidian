High-level architecture





Canvas + editing: @xyflow/react for nodes/edges, custom node renderer for person cards + per-node toolbar + handles.



Auto layout: DAGRE-based hierarchical layout over spouse clusters (connected spouse components grouped into one layout slot, split left-right inside the slot).



Persistence: IndexedDB via localforage for image blobs; app JSON index (nodes/edges/positions/transforms/group overrides) stored separately to avoid JSON size limits.



Photo UI: per-node photo upload/paste + PhotoAdjustOverlay modal (rendered via createPortal(document.body, ...)) with native non-passive wheel interception.



Export:





Cards PNGs: render a dedicated card DOM subtree for each person and convert to PNG using dom-to-image-more at high pixelRatio (3).



Tree PNG: render a dedicated printable tree DOM subtree (edges drawn in an SVG overlay + compact node visuals) and convert to PNG.



ZIP: jszip bundles cards/*.png, tree.png, data.csv, data.json, and groupings.json.

Landing/Intro

A guided tour overlay on first run (stored in app state, hasSeenTour).



Primary CTA: Export for Print.



Add person nodes





User clicks canvas pane to open PersonForm prefilled with an empty person; creates a new node at the click point.



Edit person details





Node toolbar Edit opens inline PersonForm or side panel.



Live preview updates card visuals as fields change.



Photos





Set Photo opens file input and enables clipboard paste.



For each person card: main photo + thumbnail photo each with its own persisted {xPercent,yPercent,scale} transform.



Photo adjust modal





Adjust Photo opens PhotoAdjustOverlay.



Overlay shows the card frame + draggable image; supports drag to reposition and wheel to zoom.



Wheel is intercepted with { passive: false } + preventDefault() to stop canvas zoom.



Transforms are percent-based and applied consistently in both card preview and overlay.



Relationships





Node toolbar: Add Spouse, Add Parent, Add Child.



Add Child from either spouse automatically creates parent-child edges from both spouses in that spouse cluster.



Manual edge creation: drag from connection handles for parent-child edges.



Sibling relationships are derived (optional explicit sibling edges omitted; layout uses derived adjacency only if needed).



Multi-select + align/distribute





Box selection with partial inclusion (node bounding boxes intersect selection rectangle).



Shift toggles selection state.



When >=2 nodes selected: show AlignToolbar with align/distribute actions.



Align/Distribute operates on node positions in canvas coordinates and updates ReactFlow node positions.



Groupings panel





Compute birth/marriage/death quartile-by-count groups (4 buckets, or fewer if dataset smaller).



Show computed ranges + counts.



Allow manual boundary tweaks via 3 draggable boundary controls (store overrides in app state; recompute memberships).

Data model (client)





Person (id, names, dob/dod/notes, photoMain/photoThumb with blob keys + percent transform).



Edge (id, source, target, type: parent-child | spouse, plus marriage metadata on spouse edges).



AppState (nodes, edges, spouseCluster cache, UI state, grouping overrides).

Key implementation notes (non-obvious)





Photo transform math:





Use the same origin in overlay and preview: photo is positioned at frame center; translate by percent, then scale.



Overlay wheel interception:





Add a native wheel listener on the overlay root with { passive: false } and call e.preventDefault().



Box selection partial inclusion:





For each node, compute {x,y,width,height} in canvas coords and select if rectangles intersect with selection rect.



Auto layout:





Build spouse clusters from spouse edges as connected components.



Run dagre on cluster-level nodes with parent-child edges.



After layout positions for clusters, set each person node’s x offset inside cluster (left-right split) and apply y from cluster.
